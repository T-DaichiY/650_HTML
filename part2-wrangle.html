Unit 1 — Chapter 7: Tibbles with tibble
①
📦 tibble は tidyverse 用の安全な data frame
tibble は data.frame の改良版です。Rの古い挙動で初心者が混乱しやすい部分を減らし、大きなデータでも見やすく表示します。tidyverse では、ほとんどの関数が tibble を返します。
観点data.frametibble
表示大量に表示されやすい最初の数行と画面に入る列だけ表示
型見えにくい<dbl>, <chr>, <date> などが見える
文字列古いRではfactor化されやすい勝手にfactorにしない
列抽出部分一致などが起こることがあるより厳密で安全
R — tibble basics
# data.frame を tibble に変換 > as_tibble(iris) # A tibble: 150 × 5 # Sepal.Length Sepal.Width Petal.Length Petal.Width Species # <dbl> <dbl> <dbl> <dbl> <fct> # tibble() は直前に作った列をその場で使える > tibble(x = 1:5, y = 1, z = x^2 + y)
②
✍️ tibble(), tribble(), backticks
tibble() はベクトルから表を作ります。tribble() は小さなデータをコード内で見やすく入力するための関数です。数字から始まる列名やスペースを含む列名は、backticks（` `）で囲みます。
R — create tibbles
> tribble( ~name, ~score, "Aki", 80, "Mei", 92 ) # nonsyntactic names: 変な列名は `backticks` で囲む > tb <- tibble(`2000` = "number", `:)` = "smile") select(tb, `2000`)
🌟 コーチ視点
tibble は「安全装置つきの表」です。巨大データをいきなり全部見せず、列の型も横に表示してくれるので、分析前の確認が速くなります。

Unit 2 — Chapter 8: Data Import with readr
③
📥 readr は外部ファイルを tibble として読む道具
データ分析は、Rに最初から入っているデータだけでは終わりません。自分のCSVやTSVを読み込むには readr を使います。中心は read_csv() です。
関数読むファイル使う場面
read_csv()カンマ区切り最もよく使うCSV
read_csv2()セミコロン区切り小数点にカンマを使う地域のCSV
read_tsv()タブ区切りTSVファイル
read_delim()任意の区切り文字| 区切りなど
read_fwf()固定幅古いシステム由来のファイル
R — import files
# CSVを読み込む > heights <- read_csv("data/heights.csv") # Parsed with column specification: # earn = col_double(), height = col_double(), sex = col_character() # 小さな例は inline CSV でも読める > read_csv("a,b,c\n1,2,3\n4,5,6")
④
🧹 skip, comment, col_names, na
現実のCSVは、最初にメモがあったり、列名がなかったり、欠損値が . や 999 で表されていたりします。read_csv() の引数で読み込み方を調整します。
R — common import options
# 先頭2行のメタデータを飛ばす > read_csv("meta1\nmeta2\nx,y\n1,2", skip = 2) # # で始まるコメント行を無視 > read_csv("# note\nx,y\n1,2", comment = "#") # 列名がない場合 > read_csv("1,2,3\n4,5,6", col_names = FALSE) # . を NA として読む > read_csv("a,b,c\n1,2,.", na = ".")
🌟 コーチ視点
読み込みは「ただ開く」作業ではありません。列名・型・欠損値・文字コードを正しく指定して、後の分析で壊れないようにする作業です。

Unit 3 — Chapter 8: Parsing, locale, encoding, col_types
⑤
🧪 parsing は「文字列を正しい型に変換する」こと
CSV内の値は、最初はただの文字として読まれることがあります。parse_*() は文字列を logical, integer, double, date などの正しい型へ変換する関数群です。
関数変換先例
parse_integer()整数"123" → 123
parse_double()厳密な小数"1.23" → 1.23
parse_number()文字混じりの数値"$100" → 100
parse_date()日付"2010-01-01"
parse_factor()カテゴリ想定外カテゴリを検出
R — parsing examples
> parse_integer(c("1", "231", ".", "456"), na = ".") [1] 1 231 NA 456 > parse_number("It cost $123.45") [1] 123.45 > problems(parse_integer(c("123", "abc", "123.45")))
⑥
🌏 locale と encoding：国・言語による違い
数値や日付、文字コードは国や環境によって違います。小数点が , の国、桁区切りが . の国、日本語CSVの Shift-JIS など、読み込み時に指定が必要な場合があります。
R — locale / encoding / col_types
# 小数点が , の場合 > parse_double("1,23", locale = locale(decimal_mark = ",")) [1] 1.23 # 日本語CSVが文字化けする場合の例 > read_csv("japanese.csv", locale = locale(encoding = "Shift-JIS")) # 型を明示して読み込む：再現性が高くなる > read_csv("challenge.csv", col_types = cols( x = col_double(), y = col_date() ))
⚠ readr は最初の1000行で型を推測する
最初の1000行だけ整数で、1001行目から小数が出ると、型推測が間違うことがあります。重要な読み込みでは col_types を指定しましょう。

Unit 4 — Chapter 9: Tidy Data with tidyr
⑦
🎯 tidy data の3原則
Chapter 9 は Part II の中心です。tidy data とは、変数が列、観測が行、値がセルにあるデータです。この形にすると、dplyr, ggplot2, modelr が自然に使えます。
Rule 1
変数 = 列
year, cases, population はそれぞれ列
Rule 2
観測 = 行
1国×1年など、1観測が1行
Rule 3
値 = セル
1セルには1つの値だけ
Wide → Long のイメージ
country | 1999 | 2000
→
country | year | cases
⑧
↕️ pivot_longer() / pivot_wider()
教科書では gather() と spread() が使われていますが、現在は pivot_longer() と pivot_wider() が標準です。
旧関数現在の関数意味
gather()pivot_longer()横に広がった列を縦にする。wide → long
spread()pivot_wider()縦に散らばった値を横にする。long → wide
R — pivot functions
# table4a: 1999, 2000 という列名は year の値 > table4a %>% pivot_longer( cols = c(`1999`, `2000`), names_to = "year", values_to = "cases" ) # table2: type列の値を列名に戻す > table2 %>% pivot_wider(names_from = type, values_from = count)
⑨
✂️ separate() / unite()
1列の中に複数情報が入っているなら separate()。逆に複数列を1列にまとめるなら unite() です。
R — separate / unite
# rate = "745/19987071" を cases と population に分ける > table3 %>% separate(rate, into = c("cases", "population"), sep = "/", convert = TRUE) # century と year を結合して 1999 にする > table5 %>% unite(new, century, year, sep = "")

Unit 5 — Chapter 9: Missing Values & WHO Case Study
⑩
🕳️ 明示的欠損と暗黙的欠損
欠損値には2種類あります。セルに NA があるのが explicit missing。行そのものが存在しないのが implicit missing です。tidying によって、暗黙的欠損を明示的欠損にできます。
Explicit missing
NAが見えている
2015 Q4 return = NA
Implicit missing
行が存在しない
2016 Q1 の行がない
R — complete / fill
# 全ての year × qtr の組み合わせを作る > stocks %>% complete(year, qtr) # 上の値を下へ埋める > treatment %>% fill(person)
⑪
🌍 WHO TB case study：現実データを tidy にする
WHOデータでは、new_sp_m014 のような列名の中に複数情報が埋まっています。これは現実データで非常によくある問題です。列名を値として集め、文字列を分割して tidy にします。
コード名の分解
new_sp_m014
→
new
sp
m
014
R — WHO tidying pipeline
> who %>% pivot_longer( cols = new_sp_m014:newrel_f65, names_to = "code", values_to = "cases", values_drop_na = TRUE ) %>% mutate(code = stringr::str_replace(code, "newrel", "new_rel")) %>% separate(code, c("new", "type", "sexage"), sep = "_") %>% select(-new, -iso2, -iso3) %>% separate(sexage, c("sex", "age"), sep = 1)
💡 なぜ mutate() で newrel を直すのか
new_sp などは underscore があるのに、newrel だけ underscore がないため、そのまま separate(sep = "_") すると列数が合わず失敗します。

Unit 6 — Chapter 10: Relational Data with dplyr
⑫
🔑 key で複数テーブルをつなぐ
現実の分析では、1つの表だけで完結しません。複数の表を key でつなぎます。自分の表で行を一意に識別するのが primary key、他の表を参照するのが foreign key です。
primary key
自分の表で一意
planes$tailnum は各機体を識別
foreign key
他の表を参照
flights$tailnum は planes に接続
R — check keys
# primary key が本当に一意か確認 > planes %>% count(tailnum) %>% filter(n > 1) # weather は複数列で一意 > weather %>% count(year, month, day, hour, origin) %>% filter(n > 1)
⑬
🔗 joins：基本は left_join()
left_join() は左の表を残したまま、右の表から一致する情報を追加します。分析ではこれをデフォルトにすると安全です。inner_join() は一致しない行を消すので注意が必要です。
join残す行使う場面
left_join()左を全部残す基本。追加情報を足す
inner_join()両方にある行だけ消えてよい行が明確なとき
full_join()両方全部データ統合・監査
semi_join()右に一致する左の行条件表で絞る
anti_join()右に一致しない左の行joinの問題診断
R — joins
# carrier コードから航空会社名を追加 > flights %>% left_join(airlines, by = "carrier") # 列名が違うkeyをつなぐ：dest と faa > flights %>% left_join(airports, by = c("dest" = "faa")) # planesに機体情報がない flights を調べる > flights %>% anti_join(planes, by = "tailnum")
⚠ duplicate keys に注意
両方の表で key が重複していると、全組み合わせが作られて行数が増えます。join 前に count(key) %>% filter(n > 1) で確認しましょう。

Unit 7 — Chapter 11: Strings with stringr
⑭
🔤 stringr は文字列を扱う道具
現実のデータでは、文字列の中に情報が埋まっています。stringr は、文字数を数える、結合する、切り出す、置換する、分割するための関数群です。
関数意味例
str_length()文字数"apple" → 5
str_c()結合"x" + "y" → "xy"

R — stringr basics
> str_length(c("a", "R for data science", NA)) [1] 1 18 NA > str_c(c("x", "y", "z"), collapse = ", ") [1] "x, y, z" > str_sub(c("Apple", "Banana"), 1, 3) [1] "App" "Ban"
⑮
🧩 regex は文字列パターンの言語
正規表現は、文字列のパターンを短く表す言語です。最初は暗号のように見えますが、パターンを分解すれば読めます。複雑な1つのregexより、簡単なregexを複数組み合わせる方が安全です。
regex意味例
^aaで始まるapple
a$aで終わるbanana
\\d数字0–9
[aeiou]母音のどれかa/e/i/o/u
+1回以上a+
R — regex tools
# e を含むか > str_detect(c("apple", "banana", "pear"), "e") [1] TRUE FALSE TRUE # 数字部分を取り出す > str_extract("It cost $123.45", "\\d+\\.\\d+") # 母音を全部 - に置換 > str_replace_all("apple", "[aeiou]", "-") [1] "-ppl-"
⚠ バックスラッシュは二重に書く
regexで数字を表すのは \d ですが、Rの文字列として書くと "\\d" です。ドットそのものは "\\." と書きます。

Unit 8 — Chapter 12: Factors with forcats
⑯
🏷️ factor はカテゴリ変数を順序つきで扱う仕組み
月、満足度、所得階層、政党IDのような変数は、ただの文字列ではなくカテゴリです。factor を使うと、可能な値と表示順を制御できます。
R — factors
> month_levels <- c("Jan", "Feb", "Mar", "Apr", "May", "Jun", "Jul", "Aug", "Sep", "Oct", "Nov", "Dec") x <- c("Dec", "Apr", "Jan", "Mar") factor(x, levels = month_levels) # Jam は存在しないlevelなので NA になる > factor(c("Dec", "Apr", "Jam"), levels = month_levels)
⑰
📊 forcats：グラフを読みやすくする factor 操作
forcats は factor 操作用のパッケージです。カテゴリの順番や名前を変えることで、グラフの読みやすさが大きく変わります。
関数意味使いどころ
fct_reorder()別の数値で並べ替え平均値順にカテゴリを並べる
fct_infreq()頻度順棒グラフを多い順にする
fct_relevel()特定levelを前へNot applicable を先頭へ
fct_recode()ラベル変更短いラベルを読みやすくする
fct_lump()少数カテゴリをOtherへカテゴリが多すぎる時
R — forcats
# 平均tvhours順にreligを並べる > relig %>% mutate(relig = fct_reorder(relig, tvhours)) %>% ggplot(aes(tvhours, relig)) + geom_point() # 少数カテゴリをまとめる > gss_cat %>% mutate(relig = fct_lump(relig, n = 10))
⚠ fct_recode() の向きに注意
"新しい名前" = "古い名前" の順番です。逆に書くミスが非常に多いです。

Unit 9 — Chapter 13: Dates and Times with lubridate
⑱
📅 lubridate は日付・日時を人間の感覚で扱う道具
日付は簡単そうに見えて、うるう年、月の日数、夏時間、タイムゾーンが絡むため複雑です。lubridate は、文字列から日付・日時を作り、年・月・曜日などの部品を取り出せます。
R — parse dates
# 関数名が日付の順番を表す > ymd("2017-01-31") > mdy("January 31st, 2017") > dmy("31-Jan-2017") # 日時 > ymd_hms("2017-01-31 20:11:59") # 複数列から日時を作る > make_datetime(year, month, day, hour, minute)
⑲
⏱️ components, rounding, time spans
日時から year(), month(), wday() などで部品を取り出せます。さらに、時間の長さには duration, period, interval の3種類があります。
種類考え方例
duration正確な秒数ddays(1) = 86400秒
period人間のカレンダー単位days(1) = 翌日
interval開始と終了を持つ期間start %--% end
R — components and time spans
> datetime <- ymd_hms("2016-07-08 12:34:56") year(datetime); month(datetime); wday(datetime, label = TRUE) # 週ごとに丸める > flights_dt %>% count(week = floor_date(dep_time, "week")) # 517 → 5時17分に分解 > 517 %/% 100 # 5 517 %% 100 # 17

🌐 time zones：with_tz() と force_tz()
タイムゾーンは Asia/Bangkok, America/New_York のような IANA 名で指定します。with_tz() と force_tz() は似ていますが、意味が大きく違います。
with_tz()
同じ瞬間を別表示
瞬間は同じ。表示時刻だけ変わる
force_tz()
時計表示を固定して付け替え
表示は同じ。実際の瞬間が変わる
R — time zones
> x <- ymd_hms("2026-05-17 09:00:00", tz = "UTC") with_tz(x, "Asia/Bangkok") # 同じ瞬間をバンコク表示 force_tz(x, "Asia/Bangkok") # 9:00をバンコク時間だったことにする
