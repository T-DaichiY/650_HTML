R for Data Science / PART IV — Model

このページは PART IV「Model」を、探索のためのモデリングとして整理した勉強サイトです。目的は「統計検定を完璧にする」ことではなく、 パターンsignal と 残りresiduals を分けて、データをもっと深く読むことです。

モデルは「真実そのもの」ではなく、データを理解しやすくするための単純化された近似。

PART IVの流れ：① モデルの仕組み → ② 実データでモデルを育てる → ③ 多数のモデルをまとめて扱う。

Core Message
  • 強いパターンをモデルで取り除く。
  • 残差を見ると、隠れていた弱いパターンが見える。
  • モデルは確認よりも、まず探索に使う。
  • たくさんの小さなモデルは、複雑なデータを読む強力な方法。

🧭 モデルは「探索」を強くする道具
PART IVでは、モデルを formal inference（正式な推論・検定） の道具としてではなく、 EDAの延長として使います。つまり、モデルで強いパターンを明示化し、そこから外れた部分、つまり residuals を観察して、次の仮説を作ります。
① model
強い構造を取り出す
例：価格はcaratに強く影響される
② residuals
残ったズレを見る
例：重さを除いた後、cut/color/clarityを見る
③ new signal
弱いパターンを発見
見えなかった構造が見えるようになる
⚠ 探索と確認を混ぜない
同じデータで「仮説を作る」と「仮説を証明する」を両方やると、結果を過信しやすくなります。 本当に確認したいなら、最初に training / query / test に分け、test data は最後に一度だけ使う、という考え方が重要です。

Unit 1 — Chapter 18 / Model Basics with modelr
①
📐 model family と fitted model
モデル作成は2段階です。まず model family（どんな形のパターンを探すか）を決めます。 次に、そのfamilyの中から、データに最も近い fitted model を探します。
Model family
y = a0 + a1 * x
まだ具体的な線ではない。「直線で説明する」という型。
Fitted model
y = 4.22 + 2.05 * x
実データに最も近いと判断された具体的な線。
🌟 小学生でもわかる例
model family は「どんな形の定規を使うか」。fitted model は「その定規をデータに一番合う場所に置いた結果」。直線定規を選ぶか、曲線定規を選ぶかで、見えるパターンが変わります。
②
🎚️ 直線モデルを動かして RMSE を見る
良いモデルとは、実際の点と予測線の距離が小さいモデルです。ここでは slope と intercept を動かして、 RMSE（root mean squared error） がどう変わるかを確認できます。
Interactive Model Toy
点 = data / 線 = prediction
RMSE
0.00
③
🧮 距離を小さくする：random search → grid search → optim() → lm()
Chapter 18では、モデルを当てはめる仕組みを段階的に見ます。ランダムにたくさん線を引く、gridで探す、optim()で最小化する、そして最後に線形モデル専用の lm() を使います。
方法考え方意味
random modelsランダムな係数を試す良い/悪いモデルの直感を作る
grid search係数の候補を格子状に試す最小値の場所をより体系的に探す
optim()距離を最小化する数値計算任意のモデルfamilyに使える考え方
lm()線形モデル専用の高速な解法線形モデルなら一発で最小値を出せる
R — model family, distance, lm()
# model family: y = a0 + a1*x > model1 <- function(a, data) a[1] + data$x * a[2] # distance: observed - predicted をRMSEにまとめる > measure_distance <- function(mod, data) { + diff <- data$y - model1(mod, data) + sqrt(mean(diff ^ 2)) + } # linear model: formula で family を指定 > sim1_mod <- lm(y ~ x, data = sim1) > coef(sim1_mod) (Intercept) x 4.22 2.05
④
🔎 Predictions と Residuals
モデルを見るときは係数だけを眺めるより、予測値と残差を可視化した方が直感的です。 予測は「モデルが捕まえたパターン」、残差は「モデルが見逃したパターン」です。
Prediction
add_predictions()
モデルが各xに対して予測したy。線や曲線として表示しやすい。
Residual
add_residuals()
observed - predicted。0周辺にランダムならモデルは主要パターンを捕まえた可能性が高い。
R — predictions and residuals
# predictions: 予測用のgridを作って線を描く > grid <- sim1 %>% data_grid(x) %>% add_predictions(sim1_mod) > ggplot(sim1, aes(x)) + geom_point(aes(y = y)) + geom_line(data = grid, aes(y = pred)) # residuals: 元データに残差を足して確認する > sim1 <- sim1 %>% add_residuals(sim1_mod) > ggplot(sim1, aes(x, resid)) + geom_ref_line(h = 0) + geom_point()
⑤
🧩 Formula, categorical variables, interactions
Rのモデル式 y ~ x は、単なる計算式ではなく「モデルfamilyの指定」です。 カテゴリ変数は0/1列に変換され、+ と * でモデルの意味が大きく変わります。
y ~ x
Rはこれを「切片 + xの効果」として解釈します。内部的には (Intercept) という1だけの列が入ります。
カテゴリ変数
例えば y ~ sex のようにカテゴリを使うと、Rは sexmale のような0/1列を作ります。カテゴリごとの平均を予測するモデルとして理解するとわかりやすいです。
y ~ x1 + x2
それぞれの効果を独立に足すモデルです。カテゴリ別に線を引くと、線の傾きは同じで、切片だけが違うイメージです。
y ~ x1 * x2
これは x1 + x2 + x1:x2 を意味します。交互作用を含むので、カテゴリごとに傾きも切片も変えられます。
変換
log(y) ~ log(x) や poly(x, 2) のように、非線形に見える関係を線形モデルで扱いやすくします。

Q1
model family の説明として最も正しいものは？
✅ 正解。model family は「直線」「曲線」など、探したいパターンの一般形です。
❌ model family は最終結果ではなく、モデルの型です。具体的な係数が入ったものが fitted model です。
Q2
residual の基本式はどれ？
✅ 正解。残差は実測値から予測値を引いたズレです。
❌ residual は observed - predicted です。モデルが説明できなかった部分を見ます。
Q3
y ~ x1 * x2 の意味は？
✅ 正解。* は main effects と interaction を同時に含みます。
❌ * は積だけではなく、x1 + x2 + x1:x2 の意味です。
Q4
add_predictions() の主な目的は？
✅ 正解。予測値を列として足し、線や曲線として可視化しやすくします。
❌ add_predictions() はモデルの予測値を追加する関数です。
Q5
「All models are wrong, but some are useful」の意味に近いものは？
✅ 正解。モデルは真実そのものではなく、役に立つ単純化です。
❌ モデルは完全な真実ではありません。重要なのは「役に立つ近似か」です。

Unit 2 — Chapter 19 / Model Building
①
💎 なぜ低品質ダイヤの方が高く見えるのか？
一見すると、cut/color/clarity が低いダイヤの方が高く見えます。しかしこれは confounding variable（交絡変数） である carat の影響です。 大きいダイヤほど高く、低品質のダイヤほど大きい傾向があるため、品質と価格の関係が逆に見えていました。
見かけ
低品質ほど高い？
boxplotだけ見ると逆に見える
原因
carat が強すぎる
価格を一番強く説明する変数
解決
carat効果をモデルで除く
残差で相対価格を見る
R — diamonds model building
# 1. caratとpriceをlog変換して線形に近づける > diamonds2 <- diamonds %>% + filter(carat <= 2.5) %>% + mutate(lprice = log2(price), lcarat = log2(carat)) # 2. caratだけでpriceを説明するモデル > mod_diamond <- lm(lprice ~ lcarat, data = diamonds2) # 3. residualを見ると「重さを考慮した相対的な高い/安い」が見える > diamonds2 <- diamonds2 %>% add_residuals(mod_diamond, "lresid") > ggplot(diamonds2, aes(cut, lresid)) + geom_boxplot()
📌 log2 residual の読み方
lresid = -1 は「予測価格の半分くらい」、lresid = 1 は「予測価格の2倍くらい」を意味します。log2スケールなので、残差の1単位が2倍/半分として読めます。
②
🧱 モデルを少しずつ育てる
Chapter 19の中心は、いきなり完璧なモデルを作ることではありません。 可視化 → 強いパターンをモデル化 → 残差を見る → 次のパターンを探す という反復です。

③
✈️ daily flights：曜日効果を取り除く
NYCの1日あたりフライト数は、最初は単純に見えます。しかし曜日効果が非常に強く、週末、特に土曜日は便数が少ない。 まず n ~ wday で曜日効果をモデル化し、残差を見ます。
R — daily flights model
# 1日ごとのフライト数 > daily <- flights %>% + mutate(date = make_date(year, month, day)) %>% + group_by(date) %>% + summarize(n = n()) %>% + mutate(wday = wday(date, label = TRUE)) # 曜日効果をモデル化 > mod <- lm(n ~ wday, data = daily) > daily <- daily %>% add_residuals(mod) > ggplot(daily, aes(date, resid)) + geom_ref_line(h = 0) + geom_line()
raw n
曜日パターンが強すぎる
長期トレンドや休日が見えにくい
resid
期待便数からのズレ
祝日・夏休み・季節性が見えやすくなる
④
🏖️ interaction と robust model
土曜日の便数は季節によって変わるため、wday * term のように曜日と季節の交互作用を入れます。 ただし外れ値が平均を引っ張るので、外れ値に強い MASS::rlm() も使います。
モデル意味残差で確認すること
n ~ wday曜日ごとの平均便数土曜の季節パターンが残る
n ~ wday * term曜日効果を季節ごとに変える夏休み/秋/春の違いを捕まえる
MASS::rlm()外れ値に強いロバスト回帰祝日などの極端な日で平均がズレにくい
ns(date, 5)滑らかな季節曲線データにより柔軟に語らせる

Q1
低品質ダイヤが高く見えた主な理由は？
✅ 正解。caratが価格に強く効き、低品質ダイヤほど大きい傾向があるため、品質の効果が逆に見えていました。
❌ 主因は carat です。重さの効果を除くと、品質が高いほど相対価格も高い関係が見えます。
Q2
残差を見る理由として最も適切なのは？
✅ 正解。残差はモデルが説明しきれなかった部分なので、次のパターン探しに使えます。
❌ 残差を見る目的は、モデルで取り除いた後に残る構造を読むことです。
Q3
n ~ wday はdaily flightsで何をモデル化している？
✅ 正解。曜日によって便数が違うという強いパターンをモデル化しています。
❌ wday は day of week、つまり曜日です。日ごとの便数の曜日効果を表します。
Q4
wday * term を使う理由は？
✅ 正解。交互作用により、曜日の効果がspring/summer/fallで変わるモデルになります。
❌ * は交互作用です。曜日の効果がtermによって違う可能性を入れます。
Q5

✅ 正解。robust linear model は外れ値の影響を減らすのに役立ちます。
❌ rlm() はロバスト回帰で、外れ値が推定を歪めるときに役立ちます。

Unit 3 — Chapter 20 / Many Models with purrr and broom
①
🌍 多数の小さなモデルで複雑なデータを読む
Chapter 20では、gapminderデータを使い、国ごとに lifeExp ~ year のモデルを作ります。 1つの巨大モデルではなく、国ごとの小さなモデルを142個作ることで、普通のグラフでは見えにくい国ごとの異常や変化を見つけます。
group_bycountry, continent国ごとに分ける
nestlist-column各国のデータを1セルに入れる
mapfit model各国にlmを当てる
broomtidy outputモデル結果を表に戻す
②
📦 nested data と list-column
nest() すると、1行が1観測ではなく 1グループを表す meta-observation になります。 例えば1行が「Afghanistanという国の12年分データ」を表します。これが list-column の力です。
R — nested data workflow
# 国ごとにデータをnestする > by_country <- gapminder %>% + group_by(country, continent) %>% + nest() # 各国に同じモデルを当てる > country_model <- function(df) lm(lifeExp ~ year, data = df) > by_country <- by_country %>% + mutate(model = map(data, country_model))
なぜlist-columnに入れるのか？
データ、モデル、残差、モデル評価を同じ行に保てるからです。別々のオブジェクトにすると、filterやarrangeしたときに対応関係が壊れる危険があります。
③
🧬 map(), map2(), unnest()
map() は「各要素に同じ処理をする」、map2() は「2つの対応する要素を使って処理する」、unnest() は「list-columnを普通のデータフレームに戻す」関数です。
関数使い方Chapter 20での役割
map(data, f)1つのlist-columnに関数を適用国ごとのデータにモデルをfit
map2(data, model, f)2つのlist-columnをペアで使う各国データ + 各国モデルから残差を作る
unnest(resids)入れ子データを展開残差をggplotで描ける普通の表に戻す
R — residuals for many models
# 各国データと各国モデルをペアで使ってresidualsを作る > by_country <- by_country %>% + mutate(resids = map2(data, model, add_residuals)) # list-columnを普通のデータに戻す > resids <- unnest(by_country, resids) > ggplot(resids, aes(year, resid, group = country)) + geom_line(alpha = 1/3)
④
🧹 broom：モデルをtidy dataに戻す
モデルオブジェクトはそのままでは扱いにくいので、broom で普通の表に変換します。 これにより、arrange() や filter() で「当てはまりの悪い国」を探せます。
1 model → 1 row
glance()
R², sigma, AICなどモデル全体の指標
1 coefficient → 1 row
tidy()
係数、標準誤差、統計量など
1 observation → 1 row
augment()
予測値、残差、影響度などを元データに追加
R — broom workflow
# モデルの評価指標を国ごとに取り出す > glance <- by_country %>% + mutate(glance = map(model, broom::glance)) %>% + unnest(glance, .drop = TRUE) # 当てはまりが悪い国を探す > glance %>% arrange(r.squared) # Rwanda, Botswana, Zimbabwe... などが低いR²として見つかる
⚠ R²が低い = 失敗、ではない
R²が低い国は「線形トレンドだけでは説明しにくい国」です。これは悪いニュースではなく、HIV/AIDS、紛争、社会変化などの重要な歴史的シグナルを見つける入口になります。
